🚢 船舶跟踪功能修复总结

📋 问题概述

用户报告船舶管理中的船舶跟踪功能无法获取船舶列表，页面无法正常显示船舶数据和地图标记。

🔍 问题诊断

发现的问题：

  1. TypeScript类型错误：
    • Cannot find module '@/utils/constants'
    • Cannot find module '@/api/ship'
    • Property 'AMap' does not exist on type 'Window'
  2. API调用问题：使用了可能有问题的import依赖
  3. 字段映射错误：前端期望的字段名与后端实体类不匹配
  4. 地图初始化问题：高德地图API加载和初始化流程有问题

🔧 修复方案

1. 重构页面结构

修复前：
• 使用 TypeScript (.vue文件中lang="ts")
• 复杂的import依赖
• 硬编码的模拟数据
• 分散的地图和列表布局
修复后：
• 使用 JavaScript (移除lang="ts")
• 简化的fetch API调用
• 从真实API获取数据
• 统一的响应式布局

2. API调用修复

// 修复前 - 使用复杂的import import { getAllShips } from '@/api/ship' const response = await getAllShips() // 修复后 - 使用原生fetch const response = await fetch('/api/ships/all') const result = await response.json()

3. 字段映射修复

根据后端Ship实体类的实际字段进行映射：

  • shipName → name (船舶名称)
  • shipType → typeEn (船舶类型英文)
  • flagState → flag (船旗国)
  • status：支持数字状态码 (0=停泊, 1=航行中, 2=锚泊, 3=维修)

4. 地图功能优化

  • 简化高德地图API加载流程
  • 优化标记点创建和信息窗口显示
  • 添加船舶定位和搜索功能
  • 根据船舶状态显示不同颜色的图标

✅ 修复结果

API测试结果：

  • ✅ 后端服务：运行正常 (端口8080)
  • ✅ 船舶API：返回4艘船舶数据
  • ✅ 前端代理：配置正确
  • ✅ 数据字段：映射正确

🚀 使用说明

  1. 打开浏览器访问 http://localhost:5173
  2. 进入 船舶管理 → 船舶跟踪 页面
  3. 查看左侧船舶列表，显示4艘船舶信息
  4. 查看右侧地图，船舶标记按状态显示不同颜色
  5. 使用搜索框测试筛选功能
  6. 点击列表中的"定位"按钮或地图标记查看详情

📝 技术要点

  • 前端框架：Vue 3 + Element Plus
  • 地图服务：高德地图 API 2.0
  • 数据获取：Fetch API + Vite代理
  • 状态管理：Vue 3 Composition API
  • 样式系统：Scoped CSS + 响应式布局
